Amazon Playbook

SEO y Contenido para Amazon

Checklist completo de keywords, bullet points y FAQs optimizadas para el buscador interno de Amazon (A9/A10) y máxima relevancia orgánica.

1. Research de Keywords

Identifica términos con alto volumen y baja competencia usando herramientas como Helium 10, Jungle Scout o Cerebro.

  • Long-tail primero: "zapatillas running mujer amortiguación" > "zapatillas"
  • Backend keywords: usa los 250 bytes permitidos sin repetir (Amazon ignora duplicados)
  • Sinónimos y errores comunes: "smartwatch" + "reloj inteligente" + "smart watch"
  • Seasonal/trending: incorpora términos estacionales si aplica ("verano 2026")

2. Título Optimizado

El título es el campo con más peso SEO. Estructura recomendada (máx. 200 caracteres):

Fórmula:

Marca + Palabra clave principal + Beneficio clave + Especificaciones + Diferenciador

Ejemplo: "NeoFit Zapatillas Running Mujer Amortiguación Gel | Transpirables Ligeras Talla 35-42 | Diseño Ergonómico 2026"

  • Keyword principal en las primeras 5 palabras
  • Evita keyword stuffing (penaliza A10)
  • Usa mayúsculas solo en nombres propios/marcas
  • No uses símbolos raros o emojis (contra TOS)

3. Bullet Points (5 viñetas)

Combina beneficios emocionales + features técnicas. Cada bullet debe tener keyword secundaria.

✓ Estructura ideal:

BENEFICIO EN MAYÚSCULAS: descripción técnica con keyword natural

✗ Evitar:

Listas planas de specs sin contexto emocional

Ejemplo bullet:
AMORTIGUACIÓN GEL AVANZADA: Sistema de absorción de impacto en talón y antepié para running de larga distancia sin dolor articular

4. Descripción y Contenido A+

La descripción HTML tradicional importa menos (muchos usuarios no la ven), pero A+ es clave para conversión y tiempo en página.

  • A+ Brand Story: narrativa de marca con imágenes lifestyle (aumenta conversión ~5%)
  • Comparison charts: tabla comparativa con competencia o variantes
  • FAQ module: responde objeciones comunes (tallas, materiales, garantía)
  • Storytelling visual: problema → solución → resultado con iconografía clara

5. Backend Search Terms

Campo oculto de 250 bytes (caracteres, no palabras). Amazon indexa todo sin mostrar al usuario.

✓ Incluir:

Sinónimos, errores ortográficos, plurales/singulares, términos de nicho

✗ No incluir:

Repeticiones del título, marcas de competencia (TOS), palabras vacías (el, la, de)

Truco: separa keywords con espacios, sin comas ni duplicados. Amazon tokeniza automáticamente.

6. Preguntas y Respuestas (Q&A)

Amazon indexa las Q&A. Promueve preguntas con keywords long-tail y respóndelas tú mismo.

  • Pregunta con keyword: "¿Estas zapatillas running son aptas para maratón?"
  • Respuesta con contexto: "Sí, el modelo NeoFit Pro está diseñado para distancias de 42km con amortiguación reforzada"
  • Upvotea respuestas útiles para que aparezcan primero

7. KPIs y Métricas de Seguimiento

Ranking orgánico

Trackea posición para tus keywords principales (Helium 10 Keyword Tracker)

CTR (Click-Through Rate)

% de impresiones que generan clic. Óptimo: >0.5%

Conversion Rate

% de visitas que compran. Benchmark: 10-15% para productos optimizados

Sesiones orgánicas

Tráfico sin PPC. Meta: 60%+ del total para reducir dependencia de ads

Checklist Pre-Launch

  • Título con keyword principal en primeras 5 palabras (máx 200 char)
  • 5 bullets con beneficios + keywords secundarias (máx 500 char c/u)
  • Backend search terms sin duplicados (250 bytes)
  • Descripción HTML básica (aunque A+ es prioridad)
  • Contenido A+ con 4-7 módulos (Brand Story + Comparison + FAQ)
  • Imágenes con alt text descriptivo (Amazon no lee alt, pero ayuda a conversión)
  • Categoría y subcategoría correctas (browse nodes exactos)
  • Atributos de producto completos (talla, color, material, etc.)
